SAP Business One add-on that automates invoice consolidation and LHDN submission — with built-in data validation, dynamic JSON construction, and REST API integration.

SAP Business One VB.NET LHDN Compliance REST API SAP HANA MS SQL Server
"Dynamically constructs LHDN-compliant JSON payloads from SAP document data — handling supplier, buyer, payment, shipping, cross-border, and line-item sections in a single automated pipeline."
Problem & Solution
The Problem
  • Manual invoice consolidation for LHDN submission was time-consuming and error-prone
  • Multiple invoice types (Invoice, Credit Note, Debit Note, Refund) needed separate handling
  • No audit trail or transaction logging for submitted e-invoices
  • Formatting requirements for LHDN differ between Supplier and Buyer transaction types
The Solution
  • Filters and consolidates SAP documents by date range, document type, and transaction type
  • Dynamically builds LHDN-compliant JSON from stored procedure output
  • Submits to BDO API with bearer token authentication and logs all responses
  • Updates SAP document status and writes results back to the database automatically

Key Features
01
Multi-Type Document Support
Handles Invoice, Credit Note, Debit Note, and Refund Note — with automatic document type code remapping for Buyer vs Supplier transactions.
02
Dynamic JSON Construction
JSON payload is assembled from stored procedure field mappings — supplier, buyer, payment, shipping, cross-border, and line items built programmatically.
03
Consolidation Number Tracking
Auto-generates sequential consolidation numbers (prefix + padded running number) per document type, persisted to the database after each submission.
04
Bearer Token Auth
Authenticates against the BDO API using encrypted client credentials, retrieves a bearer token, then submits the payload with full header configuration.
05
SAP Status Write-Back
On success, marks SAP documents as integrated (U_IsIntegrateToLHDN = Y) and records the consolidation number. On failure, writes the error message back to the document.
06
Dual DB Compatibility
Detects SAP HANA vs MS SQL Server at runtime — uses CALL syntax for HANA and EXEC for MSSQL, with identical business logic across both.
API Integration Detail
BDO / LHDN REST API
RestSharp · JSON.NET
Authentication
POST to auth endpoint with encrypted client credentials → returns bearer token
Submission Headers
Authorization (Bearer), clientId, onBehalfOf (TIN), isUDF — all configured from @E_API_CONFIGURATION UDO
Payload
LHDN-compliant JSON with supplier, buyer, transaction details, line items, payment, shipping, and cross-border sections
Response Handling
Parses processStatus, referenceId, successRecords, and errorDetails — routes result to SAP document update accordingly
Timeout
900,000ms configured — accommodates large consolidation batches
Local JSON Backup
Each payload saved to C drive with timestamp — full audit trail per submission

Technical Architecture
→
AIX_Consolidation

→
filterdata()
Calls SP_AI_LHDN_CONSODILATION_RETRIEVE with date range, doc type, and transaction type — populates the selection grid with matching documents.
→
SendEInvoice()
Core pipeline — executes consolidation SP, builds full JSON payload from field mappings, authenticates with BDO API, submits, and processes response.

SAP UDO storing API endpoints, client credentials, and configuration flags — no hardcoded secrets in code.
→

Writes timestamped error logs to C Drive — captures message, stack trace, source, and target site per failure.
