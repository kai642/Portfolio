SAP B1 Add-On · Solo Project

Pallet
Arrangement

Automated end-to-end warehouse packing workflow inside SAP Business One — eliminating manual pallet planning and improving space utilisation.

SAP Business One VB.NET SAP HANA MS SQL Server UDO
"Handled complex trade-offs between weight and space using multi-criteria decision logic — balancing pallet load distribution across multiple optimisation objectives simultaneously."
Problem & Solution
The Problem
  • Manual pallet arrangement was slow and inconsistent
  • No link between delivery schedules, packing, and inventory transfer
  • Weight and space trade-offs left to human judgment
  • No pack status visibility across delivery orders
The Solution
  • Converts delivery schedules directly into pallet arrangements in SAP B1
  • Auto-generates packing lists from confirmed pallet plans
  • Applies sorting and ranking algorithms to optimise load
  • Real-time pack status written back to the delivery schedule
Key Features
01
Automated Pallet Generation
Converts delivery schedule lines into pallet arrangements — respecting weight, height, and mix-compatibility from item master data.
02
Packing List Creation
Generates a Packing List UDO from confirmed pallet plans with full SO line traceability. No manual re-entry.
03
Live Delivery Schedule Picker
Reads exclusions from the in-memory datasource in real time — zero duplicate scheduling across deliveries.
04
Pack Status Tracking
Automatically updates OPEN → IN_PROGRESS → COMPLETED status on the delivery schedule as packing progresses.
05
Inventory Transfer Push
One-click push from a saved Packing List into SAP's Inventory Transfer Request — item codes, quantities, and SO references auto-filled.
06
Dual DB Compatibility
Auto-detects SAP HANA vs MS SQL Server at runtime — deployable on either database with no code changes.
Algorithms & Logic
Optimisation Engine
Technical Depth
Bubble Sort
Ordering / Priority
Orders items by weight and priority before pallet assignment. Heavier items placed first for a stable load base — preventing top-heavy arrangements during transit.
Non-Dominated Sorting
Multi-Criteria Ranking
Rank-based approach that balances weight, space, and mix constraints simultaneously — items non-dominated across all criteria are ranked first, mirroring multi-objective optimisation (NSGA-II style) for consistently balanced pallet loads.

Module Architecture
→
XCR_ORDR2
Parent orchestrator — injects buttons into the Delivery Schedule form and manages all child form lifecycle.
→
XCR_ORDR2_Form
Sales Order selector. Builds live SQL exclusion from the in-memory datasource — already-scheduled items never re-appear.
→
XCR_DeliverySchedule
Delivery schedule picker. Loads non-completed schedules and populates the pallet pack grid on import.
→
XCR_PalletArrangement
Core planning engine. Applies sorting and ranking logic, stages deferred save payload for atomic commit after Packing List Add.
→
PACKING
Packing List handler. Commits pallet writes, resolves SO references, updates delivery schedule status, and closes the arrangement form.
